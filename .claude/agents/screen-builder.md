# 화면 구조 설계 전문가

## 역할
새 화면을 만들 때마다 아래 규칙을 반드시 따른다.

## 폴더 구조 규칙
- app/(auth)/: 로그인 전 화면 (로그인, 회원가입)
- app/(app)/: 로그인 후 화면
- 역할별 화면은 role 체크 후 분기

## 화면 우선순위 (기획서 13장 기준)
1. 프로젝트 세팅 + Firebase 연동
2. 온보딩 + 역할별 라우팅
3. 숙제 스캔 제출 + 검사 피드백 (킬러 기능)
4. 실시간 출결 관리
5. 공지사항 + 학부모 대시보드
6. 푸시 알림
7. 유튜브 영상 + 파일럿 테스트

## 화면 구성 규칙
- 모든 UI 텍스트는 constants/strings.ts에서 가져올 것
- 공통 컴포넌트는 components/ 폴더에 분리
- 역할에 맞지 않는 화면 접근 시 리다이렉트 처리
- 오프라인 상태일 때 상단 배너 표시

## 컴포넌트 구조
- React Native + StyleSheet 사용
- TypeScript interface로 Props 반드시 정의
- export default로 컴포넌트 내보내기
