# 웹 대시보드 전문가

## 역할
웹 대시보드 관련 코드를 작성할 때마다 아래 규칙을 반드시 따른다.

## 기술 스택
- React + Vite + TypeScript
- TailwindCSS (스타일링)
- React Query (Firestore 데이터 페칭·캐싱)
- Recharts (출결 통계 차트)
- SheetJS (엑셀 업로드·다운로드)
- Firebase Hosting 또는 Vercel 배포

## 웹 전용 규칙
- 웹 대시보드는 admin 전용 — 다른 역할 접근 차단
- 앱과 동일한 Firebase 프로젝트 공유 — 별도 DB 생성 금지
- Apple 로그인 웹 미지원 — 이메일·카카오·Google만 허용
- Pro 플랜 가입 시에만 웹 대시보드 접근 가능 (무료 플랜 차단)
- 앱 내에서 웹 결제 페이지로 유도 금지 (앱스토어 정책 위반)

## 주요 화면 구성
1. 홈 대시보드: 오늘 출결 현황, 반별 출석률 차트, 최근 7일 추이 그래프
2. 출결 관리: 날짜별 필터, 월별 엑셀 다운로드
3. 학생·반 관리: 학생 목록 테이블, 엑셀 일괄 업로드
4. 선생님 관리: 목록 + 담당 반 현황, 초대 링크 발송
5. 공지사항 관리: 작성·수정·삭제, 읽음 현황
6. 구독·결제 관리: 플랜 확인, 웹 직접 결제 (수수료 2~3%)

## 엑셀 일괄 업로드 규칙
- 필수 컬럼: 이름 / 생년월일 / 보호자 연락처 / 반 이름 / 수강 시작일
- 업로드 후 미리보기 확인 단계 필수
- 유효성 검사: 필수값 누락·번호 형식 오류 행 하이라이트
- 완료 후 성공 N명 / 실패 N명 + 실패 사유 표시

## 웹 개발 순서
1. 프로젝트 세팅 + Firebase 연동 + 로그인
2. 홈 대시보드 — 출결 현황
