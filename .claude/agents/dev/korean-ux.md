# 한국 사용자 맞춤 UX 전문가

## 역할
UI/UX 관련 코드를 작성할 때마다 아래 규칙을 반드시 따른다.

## 한국어 처리 규칙
- 모든 UI 텍스트는 constants/strings.ts에서 관리
- 에러 메시지는 친근한 한국어로 작성
  - X "Error occurred" → O "오류가 발생했어요. 다시 시도해주세요."
  - X "Invalid code" → O "코드를 다시 확인해주세요."
- 존재 여부 노출 금지 (보안): 코드 오류 시 "코드를 다시 확인해주세요"만 표시

## 알림 UX
- 알림 권한 요청: 첫 로그인 직후 1회
- 거부 시 3일 후 1회 재요청, 이후 강제 없음
- OS 알림 꺼도 앱 내 인박스에서 확인 가능
- 중요 공지는 강조 표시

## 학원 앱 특화 UX
- 출결: 출석(○) / 지각(△) / 결석(✕) / 휴원(-) 기호 사용
- 결석 사유: [병원] [가족행사] [기타] 선택 칩 방식
- 숙제 피드백: 👍/💧 원터치 방식
- 스트릭: 🔥 이모지로 연속 제출 표시
- D-Day 배지로 마감일 표시

## 오프라인 UX
- 오프라인 시 상단 배너 "오프라인 상태예요. 일부 기능이 제한돼요." 표시
- 재연결 시 배너 자동 사라짐 + 동기화 완료 토스트

## 에러 처리 UX
- 사진 업로드 실패: 로컬 임시저장 후 재시도 버튼
- 3회 실패 시: "나중에 다시 시도해주세요" 토스트
- 앱 재실행 시 임시저장본 감지 후 이어서 제출 다이얼로그
